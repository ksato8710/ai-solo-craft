**創設者：** Moksh Garg（元プロダクトデザイナー）、Chirag Singla（ML エンジニア）  
**調達額：** $2.25M シードラウンド（2024年10月、Kalaari Capital リード）  
**利用者規模：** 500+チーム、2,500+週間アクティブユーザー

## 🔍 Figrが解決する根本問題

### デザインプロセスの非効率性

従来のプロダクト開発では、デザイナーの**60-70%の時間**が非創造的作業に消費されていました：

- **リサーチ作業：** 類似プロダクトの調査、UXパターンの収集
- **反復作業：** ワイヤーフレーム調整、デザインシステムの構築
- **コンテキスト理解：** プロダクト要件の把握、エッジケース発見

Figrは、これらの「グランドワーク」をAIが自動化し、デザイナーを創造的作業に集中させます。

## 🚀 Figrの革新的機能

### 1. プロダクト文脈理解

**入力可能なコンテキスト：**
- スクリーンショット、Figmaファイル
- 画面録画、ウェブキャプチャ
- プロダクトドキュメント
- アナリティクスCSV
- ナレッジベース

**AI処理プロセス：**
```
プロダクト情報 → 文脈理解 → エッジケース発見 → UXフロー設計 → プロトタイプ生成
```

### 2. データドリブンUX決定

**実例：B2B SaaS トライアル最適化**
```
現状分析: 14日間トライアル転換率 8.3%
業界平均: 18.7%
AIが発見した問題:
- Step 2: 価値提供前のメール認証 → 34%離脱
- Step 5: 初期チーム招待必須 → 41%未完了
- Step 7: インテグレーション設定必須 → 28%放棄

AI提案:
- 認証を価値体験後に移動
- チーム招待を3日後に延期
- インテグレーション設定に「後で」オプション
```

### 3. 実績あるUXパターンの活用

**参照データベース：**
- **100,000+** 実証済みデザインパターン
- Zoom、X.com、Wise、Waymo等の実例
- 業界別ベストプラクティス集
