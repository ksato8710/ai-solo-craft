## はじめに

### Claude Codeの特徴

Claude Codeは「ターミナルで動くAI開発パートナー」です。従来のコード補完ツールとは異なり、**エージェント**として振る舞います。

| 特徴 | 説明 |
|------|------|
| **エージェント型** | 指示を解釈し、計画を立て、複数のツールを使って自律的に作業 |
| **コードベース理解** | プロジェクト全体を把握した上で、適切なファイルを特定・編集 |
| **ターミナル実行** | シェルコマンドの実行、結果の解釈、エラーへの対応 |
| **Git統合** | コミット、ブランチ作成、PR作成まで一貫して対応 |
| **マルチ環境** | ターミナル、VS Code、JetBrains、デスクトップ、Webで利用可能 |

### なぜClaude Codeなのか

Claude Codeの強みは**コンテキスト管理とツール呼び出しの精度**にあります。

大きな機能を実装する際、Claude Codeは：
1. 計画を立てて`plan.md`を作成
2. ToDoリストで進捗を管理
3. タスクを一つずつ実行し、完了をチェック
4. 関連ファイルの更新も自動で検知して対応

この「再帰的な自己管理」が、他のツールとの大きな違いです。

## インストールと初期設定

### インストール方法

**macOS / Linux / WSL:**

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

**Windows PowerShell:**

```powershell
irm https://claude.ai/install.ps1 | iex
```

**Homebrew（macOS）:**

```bash
brew install --cask claude-code
```

**npm経由:**

```bash
npm install -g @anthropic-ai/claude-code
```

### 初回起動

プロジェクトディレクトリで`claude`コマンドを実行：

```bash
cd your-project
claude
```

初回起動時にログインを求められます。認証方法は2つ：

| 方法 | 特徴 |
|------|------|
| **Claude Pro/Maxアカウント** | 月額固定、無制限に近い利用が可能 |
| **Anthropic API** | 従量課金、使った分だけ支払い |

**おすすめ**: 月額$20のProプランから始めて、制限に達するようであればMaxプラン（$100/月）へ移行。

### VS Code拡張機能

VS Codeユーザーは拡張機能も利用可能：

1. 拡張機能ビュー（`Cmd+Shift+X`）で「Claude Code」を検索
2. インストール後、コマンドパレット（`Cmd+Shift+P`）から「Claude Code: Open」

VS Code版では**インラインdiff**、**@メンション**、**プランレビュー**が使えます。

### JetBrains IDE

IntelliJ IDEA、PyCharm、WebStormなどJetBrains IDEでも利用可能：

1. [Claude Code plugin](https://plugins.jetbrains.com/plugin/27310-claude-code-beta-)をマーケットプレイスからインストール
2. IDEを再起動

## 3つのチャットモード

Claude Codeには3つのチャットモードがあり、`Shift+Tab`で切り替えられます。

### 1. Default Mode（デフォルト）

- Claudeが変更を**提案**し、ユーザーの**承認後**に実行
- 変更内容を確認しながら進めたい場合に最適
- `/output-style`でClaudeに理由を説明させることも可能

### 2. Auto Mode（自動）

- Claudeが**承認なしで**ファイルを編集
- 「バイブコーディング」に最適 — 指示を出してコーヒーを取りに行ける
- セキュリティ上重要なコマンド（`npm install`など）は引き続き確認を求める

**パーミッション設定:**

```bash
# 特定コマンドを自動許可
/permissions
# ワイルドカードパターンも使用可能
# Bash(npm *)
# Bash(* install)
```

**完全自動（上級者向け）:**

```bash
claude --dangerously-skip-permissions
```

⚠️ Git連携とチェックポイントを設定した上での使用を推奨。

### 3. Plan Mode（計画）

- 実装前に**Extended Thinking**で包括的な計画を立てる
- 新機能、複雑なリファクタリング、新プロジェクト開始時に使用
- 計画は`Plan.md`として保存され、セッションを跨いで参照可能

**Plan Modeの活用例:**

```
個人の家計管理アプリを作りたいです。要件：

- モダンでクリーンなUI（NotionとMint.comを参考に）
- 収入・支出・貯蓄目標の追跡
- 美しいチャートとインサイト
- React + シンプルなバックエンド
- 高速で使いやすい体験
```

Claudeは明確化のための質問をし、その後包括的な計画を提示します。フィードバックを伝えて計画を修正し、満足したら「実行して」と指示。

## プロジェクトメモリ

### CLAUDE.mdとは

`CLAUDE.md`は**プロジェクトの記憶ファイル**です。プロジェクトルートに配置すると、Claude Codeは毎セッション開始時にこのファイルを読み込みます。

### 初期化

```bash
/init
```

既存プロジェクトでこのコマンドを実行すると、Claudeがコードベースを分析して`CLAUDE.md`を自動生成します。

### CLAUDE.mdの構造例

```markdown
# CLAUDE.md

## アーキテクチャ
Next.js 14のフルスタックアプリケーション
- **フロントエンド**: React + Tailwind CSS
- **バックエンド**: API Routes + Prisma
- **データベース**: PostgreSQL（Supabase）
- **認証**: NextAuth.js

## コーディング規約
- コンポーネントは `src/components/` に配置
- 型定義は `src/types/` に配置
- 関数はアロー関数を使用
- エラーハンドリングには必ずtry-catchを使用

## 重要な決定事項
- 状態管理はZustandを採用（2026-01-15）
- i18nはnext-intlを使用（2026-01-20）
```

### 階層構造

大規模プロジェクトでは複数の`CLAUDE.md`を配置できます：

```
~/.claude/CLAUDE.md           # グローバル（個人の好み）
~/projects/CLAUDE.md          # 組織/チーム標準
~/projects/my-app/
├── CLAUDE.md                 # プロジェクト全体
├── backend/
│   └── CLAUDE.md             # バックエンド固有
├── frontend/
│   └── CLAUDE.md             # フロントエンド固有
└── docs/
    └── CLAUDE.md             # ドキュメント規約
```

より具体的なファイルが一般的なファイルを上書きします。

### 動的な更新

チャット中に`#`コマンドでCLAUDE.mdに追記できます：

```
# API呼び出しを行うコンポーネントには必ずエラーバウンダリを使用すること
```

機能が完成したら、Claudeにドキュメント更新を依頼することも推奨：

```
今実装した認証機能についてCLAUDE.mdを更新して
```

## コンテキスト管理

### コンテキストウィンドウとは

コンテキストウィンドウは、Claudeが一度に考慮できるテキストの最大量（トークン数で測定）です。

チャット履歴、読み込んだファイル、生成したコード、ツール結果のすべてがコンテキストウィンドウを消費します。

### コンテキスト管理のベストプラクティス

1. **スコープを絞る**: 1つのチャットを1つの機能/プロジェクトに限定
2. **完了後にクリア**: `/clear`でコンテキストをリセット
3. **再開が必要なら**: `/resume`で過去のセッションに戻る
4. **大規模タスク**: 計画を`plan.md`に保存し、分割して実行

### 部分的な圧縮

コンテキストが不足しそうだが完全にクリアできない場合：

```
/compact 認証実装とデータベーススキーマの決定を保持して
```

### セッション管理コマンド

| コマンド | 説明 |
|----------|------|
| `/clear` | 新しいセッションを開始 |
| `/resume` | 過去のセッションを選択して再開 |
| `/rename` | 現在のセッションに名前を付ける |
| `/stats` | 使用量グラフ、連続使用日数、セッション履歴 |

## サブエージェント

### サブエージェントとは

サブエージェントは、独自の指示、コンテキストウィンドウ、ツール権限を持つ**専門AIアシスタント**です。

```
あなた ↔ メインClaude（コーディネーター）
         ├── コードレビュアー（品質担当）
         ├── テストエンジニア（テスト担当）
         └── ドキュメントライター（技術文書担当）
```

### サブエージェントのメリット

- **コンテキスト分離**: サブエージェントの会話がメインチャットを圧迫しない
- **専門化**: 役割に特化した詳細な指示を設定可能
- **並列処理**: 複数のエージェントが同時に作業可能

### サブエージェントの作成

```
/agents
```

指示に従って種類を選択。作成されたエージェントは`.claude/agents/`フォルダにMarkdownファイルとして保存されます。

### サブエージェント定義の例

```markdown
---
name: code-reviewer
description: コード品質と保守性の包括的分析
tools: read, grep, diff, lint_runner
---

あなたは熟練のコードレビュアーです：

## レビュー優先順位（順番に）:
1. **ロジックエラーとバグ** — システム障害を引き起こす可能性
2. **セキュリティ脆弱性** — データ保護の問題
3. **パフォーマンス問題** — ユーザー体験への影響
4. **保守性の問題** — 技術的負債の増加
5. **コードスタイルと一貫性** — プロジェクト標準との整合

重要: アクションが必要な重大な問題のみ報告すること。
具体的で実行可能な改善提案を提供すること。
```

### 非同期サブエージェント

サブエージェントは**バックグラウンド実行**が可能です：

1. サブエージェントが作業中に`Ctrl+B`を押す
2. サブエージェントはバックグラウンドで継続
3. 完了時または注意が必要な時にメインスレッドに通知

これにより真の並列開発が可能：

- コードレビューのサブエージェントをバックグラウンドで起動
- メインチャットで機能実装を継続
- レビュー完了時に通知を受け取る

## Git連携とチェックポイント

### なぜGitが重要か

Claude Codeが誤って重要なコードを壊すリスクを軽減するため、Git連携は必須です。

### ブランチ戦略

1. **新機能開始時**: 新しいブランチを作成

```
新しいブランチ feature/auth-system を作成して
```

2. **実装完了後**: テストを実行

```
テストを実行して結果を確認して
```

3. **問題なければ**: コミット

```
変更をコミットして。コミットメッセージは日本語で
```

4. **機能完成後**: mainにマージ

```
mainブランチにマージして
```

### Git Worktrees（上級）

Worktreesを使うと、複数のブランチを同時に別ディレクトリでチェックアウトできます：

```bash
# メインリポジトリ
cd my-project

# 認証機能用のworktree
git worktree add ../my-project-auth -b feature/auth

# レポート機能用のworktree
git worktree add ../my-project-reports -b feature/reports
```

各worktreeで別々のClaude Codeインスタンスを起動し、並列開発が可能。失敗しても他のworktreeに影響しません。

### チェックポイント（Rewind）

Claudeが予期せぬ変更をした場合：

```
/rewind
```

送信したメッセージの一覧が表示され、任意の時点に戻れます。

## カスタムスラッシュコマンド

### コマンドフォルダの作成

```bash
mkdir -p .claude/commands
```

### コマンドファイルの作成

`.claude/commands/review.md`:

```markdown
最近の変更に対して包括的なコードレビューを実行:

1. TypeScriptとReactの規約に従っているか確認
2. 適切なエラーハンドリングとローディング状態があるか検証
3. アクセシビリティ基準を満たしているか確認
4. 新機能に対するテストカバレッジをレビュー
5. セキュリティ脆弱性をチェック
6. パフォーマンスへの影響を評価
7. ドキュメントが更新されているか確認

発見した新しいパターンでCLAUDE.mdを更新すること。
```

### コマンドの使用

```
/review
```

### チームでの共有

`.claude/commands/`フォルダをリポジトリにコミットすると、チーム全員が同じコマンドを使用できます。

### Claudeに作成させる

```
/branch というコマンドを作って。実行するとgit statusを確認し、
問題なければ新しいブランチを作成して移動する機能
```

## MCP

### Model Context Protocolとは

**MCP（Model Context Protocol）** は、AIアシスタントを外部ツールやデータソースに接続するためのAnthropicのオープンスタンダードです。

Jira、GitHub、Slack、データベースなど、あらゆるシステムとClaude Codeを連携させる「ユニバーサルコネクタ」です。

### MCPサーバーの追加

```bash
# Brave検索を追加
claude mcp add brave-search -s project -- npx @modelcontextprotocol/server-brave-search
```

### MCPステータスの確認

```
/mcp
```

### 人気のMCPサーバー

| サーバー | 機能 |
|----------|------|
| **Brave Search** | Web検索 |
| **GitHub** | リポジトリ操作、PR、Issue |
| **Supabase** | データベースアクセス |
| **Puppeteer** | ブラウザ自動操作 |
| **Slack** | メッセージ送受信 |
| **Google Drive** | ドキュメントアクセス |

### カスタムMCPサーバー

独自のMCPサーバーを構築することも可能。社内ツールや独自APIとの連携に活用できます。

## Claude Skills

### Skillsとは

**Skill**は、Claude Codeが繰り返し実行できる命令とコードのセットです。カスタムスラッシュコマンドより強力で、メタデータからClaudeが自動的に適用タイミングを判断します。

### Skillsのメリット

- **コンテキストを消費しない**: MCPのように常時ロードされない
- **自動トリガー**: メタデータに基づいて自動適用
- **再利用可能**: パッケージとして共有可能

### Skillの作成

```
/skills
```

### 活用例

- コードをプッシュする際に自動でSlack通知を送信
- 特定のファイルタイプを編集する際に専用のリンターを実行
- PRを作成する際に自動でテンプレートを適用

## プラグインシステム

### プラグインとは

**プラグイン**はSkillsの次の進化形です。マーケットプレイスからインストール可能なパッケージ化された拡張機能で、以下をバンドルできます：

- カスタムコマンド
- 専門エージェント
- Hooks
- MCPサーバー

### プラグインの管理

```
/plugins                    # プラグイン一覧を閲覧
/plugins install typescript-lsp  # インストール
/plugins list               # インストール済み一覧
/plugins update             # 全プラグインを更新
```

### 公式マーケットプレイス

Anthropicは2025年12月に36個のキュレーションされたプラグインで公式マーケットプレイスを開始。カテゴリ別に整理されています：

- コードインテリジェンス
- デプロイメントヘルパー
- ワークフロー自動化

### 注目のプラグイン

- **frontend-design**: フロントエンドデザイン支援
- **ralph-wiggum**: 自律的なループ実行（夜間作業用）
- **typescript-lsp**: TypeScript型チェック

## Hooks

### Hooksとは

**Hooks**は、Claude Codeのライフサイクルの特定ポイントで自動実行されるシェルコマンドです。Claudeが「覚えておく」ことに依存しない、**確実な自動化**を提供します。

### Hookの種類

| Hook | タイミング |
|------|----------|
| **PreToolUse** | ツール実行前 |
| **PostToolUse** | ツール実行成功後 |
| **Notification** | 通知送信時 |
| **Stop** | タスク完了時 |
| **Sub-agent Stop** | サブエージェント完了時 |

### Hookの設定

```
/hooks
```

### 活用例

- **PostToolUse（ファイル書き込み後）**: 自動フォーマット、lint実行
- **Stop**: ドキュメント自動更新、Slack通知
- **PreToolUse（コミット前）**: テスト実行、セキュリティチェック

## テスト戦略

### Claudeにテスト基盤を構築させる

```
家計管理アプリに堅牢なテストを設定して：

- ユニットテスト: 全ユーティリティ関数（通貨フォーマット、日付計算、バリデーション）
- コンポーネントテスト: React Testing Libraryで全UIコンポーネント
- 統合テスト: DBセットアップ/ティアダウンを含むAPIエンドポイント
- E2Eテスト: トランザクション追加、レポート表示などの重要フロー
- パフォーマンステスト: データ増加時の速度維持を確認

テスト基盤を構築し、既存機能の包括的テストを作成して。
変更が何も壊さないことを確信したい。
```

Claudeは：
1. 必要なパッケージをインストール・設定
2. テストユーティリティを作成
3. ビジネスロジックを反映したテストを生成

## CI/CD構築

### GitHub Actionsの自動構築

```
家計管理アプリ用の堅牢なCI/CDパイプラインを構築して：

プルリクエスト時:
- 全テストスイート実行（unit, integration, E2E）
- TypeScriptコンパイル確認
- Prettierでフォーマットチェック
- ESLintで品質チェック
- 本番バンドルのビルド確認
- 依存関係のセキュリティ監査
- 破壊的変更のチェック

mainブランチマージ時:
- 上記すべて
- ステージング環境に自動デプロイ
- スモークテスト実行
```

## マルチプラットフォーム

### Web版（claude.ai/code）

- ローカル設定不要でブラウザから利用
- GitHubリポジトリを仮想サンドボックスにクローン
- **真の自律コーディング** — タスクを依頼して完了を待つだけ
- 完了後にPRを作成してレビュー

### モバイル版（iOS）

- 外出先からバグ修正を依頼
- 完了通知を受け取りコードをレビュー
- PRをマージして本番にデプロイ

### デスクトップ版

- ローカル作業とクラウド作業の両方に対応
- ビジュアルdiffレビュー
- 複数セッションの並列実行

### Teleport機能

クラウドとローカルのセッションを移動：

**クラウドにタスクを送る:**

```
& 認証モジュールをリファクタリングして
```

`&`で始めるとWeb上で実行され、ローカルで作業を続けられます。

**クラウドからローカルに取り込む:**

```bash
claude --teleport session_abc123
```

または`/tasks`で実行中のバックグラウンドセッションを確認し、`t`でteleport。

## ベストプラクティス

### 1. 計画から始める

大きな機能は**Plan Mode**で開始。計画を`plan.md`に保存し、段階的に実行。

### 2. コンテキストを管理する

- 1チャット = 1機能
- 完了したら`/clear`
- 大規模タスクは分割

### 3. CLAUDE.mdを育てる

- `/init`で初期化
- 機能完成ごとに更新
- チームで共有

### 4. Gitを活用する

- 機能ごとにブランチ
- こまめにコミット
- `/rewind`で巻き戻し可能

### 5. 自動化を構築する

- Hooksで確実な自動化
- CI/CDでデプロイ自動化
- カスタムコマンドでワークフロー標準化

### 6. サブエージェントを活用する

- 専門タスクは専門エージェントに
- 非同期実行で並列作業
- コンテキスト分離で効率化
