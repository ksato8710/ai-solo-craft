「3週間で開発、6週間で$10k MRR」

この数字を聞いて、多くのソロ開発者は「運が良かっただけでは？」と思うだろう。しかし、**Sleek**の成功には明確な戦略がある。

この記事では、AIモバイルアプリデザインツール「Sleek」の創業者**Mattia Pomelli**の戦略を一次ソースに基づいて分析し、ソロ開発者が再現できる具体的な教訓を抽出する。

---

## この記事で得られること

- Sleekが成功した5つの戦略的要因
- 「過去の資産を再利用する」具体的な方法
- マーケティング費用ゼロでバイラルを起こすXの活用法
- フリーティアの制限設計が収益化に与える影響
- ソロ開発者が明日から実践できる3つのアクション

## Sleekとは

**Sleek**（sleek.design）は、AIを活用したモバイルアプリデザインツールだ。

ユーザーはアプリのアイデアをテキストで入力するだけで、AIが数分でモバイルアプリのモックアップを生成する。生成されたデザインは、そのままコードにエクスポートしたり、Figmaで編集したりできる。

| 項目 | 内容 |
|------|------|
| **製品名** | Sleek |
| **URL** | [sleek.design](https://sleek.design) |
| **カテゴリ** | AIデザインツール（モバイルアプリ特化） |
| **ローンチ** | 2025年11月 |
| **$10k MRR達成** | ローンチから約6週間後 |
| **マーケティング費用** | $0 |

## 創業者プロフィール：Mattia Pomelli

Mattia Pomelliは、8年のコーディング経験を持つ開発者であり、デザインにも情熱を注いでいる。

> 「コーディングを8年やってきて、ずっとデザインにも興味があった。最終的に、デザインツールを作ることが自分の好きなことの交差点だと気づいた」
> — Mattia Pomelli

**重要なのは、Sleekが彼の最初のプロダクトではないこと**だ。Sleek以前に、2人の友人と共に4つのデザインツールを開発している。この「蓄積」が、3週間という短期開発を可能にした。

### チーム構成

- **Mattia Pomelli** — 開発・マーケティング
- **2人の共同創業者** — 友人として参加

3人の小規模チームだが、全員が過去のプロジェクトで協働経験がある。

## 成功要因分析：5つの戦略

### 1. 過去の資産を再利用する（Code Reuse戦略）

Sleekの3週間開発は「ゼロからのスタート」ではなかった。

> 「幸運なことに、私たちはゼロから始めていなかった。以前にデザインツールを作っていたので、既存のプロダクトをモバイルデザイン用に再利用するのに3週間ほどで済んだ」
> — Mattia Pomelli

**具体的に再利用したもの:**
- UI基盤のコンポーネント
- AI生成ロジック
- 決済・認証の実装
- デプロイ・インフラ設定

**教訓:** ソロ開発者が意識すべきは「次のプロダクトで使えるか？」という視点だ。

- コンポーネントは汎用的に設計する
- 認証・決済は一度作れば使い回せる
- インフラ設定（Vercel、Supabase等）はテンプレート化

### 2. 成長市場×ニッチ問題の交差点を狙う

Sleekがターゲットにしたのは「モバイルアプリを作りたいが、デザインスキルがない開発者」だ。

> 「モバイルアプリを作る人が増えている。良いデザインは重要だが、Webと比べてこの領域のツールは少ない。だから、デザインツールを作るノウハウを活かして、モバイル特化の製品を作るチャンスがあると思った」
> — Mattia Pomelli

**市場分析:**

| 要素 | 評価 |
|------|------|
| 市場成長 | ◎（ノーコード/ローコードでアプリを作る人が増加） |
| 競合 | ○（Webデザインツールは多いが、モバイル特化は少ない） |
| 痛みの強さ | ◎（デザインがないと開発が進まない） |
| 解決の明確さ | ◎（AI生成で即座にモックアップが手に入る） |

**教訓:** 「強い問題 × 成長市場」を見つけること。

Mattiaのアドバイス:
> 「"Nice-to-have"なプロダクトを作る人が多すぎる。それは本当に売りにくい。理想は、人々のお金か時間を大幅に節約すること。さらに良いのは、収入を増やす手助けをすること」

### 3. Xバイラル戦略（$0マーケティング）

Sleekは**マーケティング費用ゼロ**で$10k MRRを達成した。その中心にあったのがXでのバイラル戦略だ。

**バイラル投稿の構造:**

1. **強いフック** — 問題を端的に提示
2. **魅力的なデモ動画** — 実際の生成プロセスを見せる
3. **アクセス獲得のためのコメント依頼** — 「comment for early access」

> 「Xのアルゴリズムはコメントを好む。価値あるコンテンツ + 強いエンゲージメント = バイラルの可能性が高い」
> — Mattia Pomelli

**実績:**
- @mattiapomelli のローンチ投稿: **846K views以上**
- コメント数: **6,500以上**
- リツイート: **約500**

**継続的なコンテンツ戦略:**

ローンチ後も、Mattiaは毎日Xに投稿している。ポイントは**製品を直接宣伝しないこと**。

> 「製品を直接プロモートするのではなく、人々にとって本当に価値がある/面白いコンテンツを作って、間接的に製品につなげる」
> — Mattia Pomelli

**具体例:**
- 「あなたのアプリアイデアをコメントしてください。Sleekでデザインします」
  - → 無料でデザインを提供し、編集したい場合はSleekを使ってもらう
- AIで作ったクールなデザインを共有（Sleekの名前を出さない）
  - → 「何のツール？」と聞かれたら回答

**教訓:** 製品宣伝ではなく「価値提供」がバイラルを生む。

### 4. フリーティアの戦略的制限

Sleekのフリーティアは**意図的に厳しく制限**されている。

| プラン | 内容 |
|--------|------|
| **無料** | AI生成1回のみ（編集・追加生成は有料） |
| **有料** | 月額/年額サブスク、AIクレジット付き |

> 「ブートストラップしている以上、無料ユーザーに高コストをかける余裕はない。Sleekでは、ユーザーは1回だけ無料で生成でき、編集や追加生成にはアップグレードが必要」
> — Mattia Pomelli

**この制限のメリット:**

1. **AIコストの抑制** — 無料ユーザーによるコスト負担を最小化
2. **顧客品質の向上** — 「価値を理解した人」だけが有料化
3. **明確な価値提示** — 1回の生成で製品の力を実感できる

**価格差別化の実験:**

> 「インドネシアやアフリカのクリエイターによるバイラル動画があり、これらの国からのトラフィックが増えた。購買力が違うので、これらの国向けに安くて制限の多いプランを試した。これで、失われていたかもしれない収益を得られた」
> — Mattia Pomelli

**教訓:** フリーティアは「試食」であり「食べ放題」ではない。

### 5. ICP（理想顧客像）の明確化

過去のプロダクトでの失敗から、Sleekでは最初からICPを明確にした。

> 「以前のツールでは、最初から誰が最も恩恵を受けるかを考えずにプロダクトを作るという典型的なミスを犯した。結果、Indie Hacker、スタートアップ、デザイナー、エージェンシー、PM…と、いろんな人が使っていた。プロダクトは誰にとっても"まあまあ"だったが、誰にも特化していなかった」
> — Mattia Pomelli

**Sleekの明確なICP:**

| 属性 | 内容 |
|------|------|
| **職種** | アプリ創業者、開発者 |
| **課題** | デザインスキルがない/デザイナーを雇う予算・時間がない |
| **求める解決** | 素早くプロ品質のモックアップを作りたい |
| **価値観** | スピード重視、「バイブデザイン」を楽しむ |

**ICPを明確にするメリット:**
- どの機能を優先するかが決まる
- 製品の説明（メッセージング）がブレない
- どこでマーケティングすべきかが明確

**教訓:** 「誰にでも売れる」は「誰にも売れない」と同義。

## ソロ開発者が明日から実践できる3つのアクション

### 1. 「再利用前提」でコードを書く

次のプロダクトで使えるように、今日から意識を変える。

**具体的なアクション:**
- 認証・決済のロジックを汎用モジュール化
- UIコンポーネントをStorybookで管理
- 環境変数とデプロイ設定をテンプレート化

### 2. 「強い問題 × 成長市場」を探す

「Nice-to-have」ではなく「Must-have」を見つける。

**チェックリスト:**
- [ ] ユーザーのお金を節約できるか？
- [ ] ユーザーの時間を節約できるか？
- [ ] ユーザーの収入を増やせるか？
- [ ] その市場は成長しているか？
- [ ] 競合は飽和していないか？

### 3. 「価値提供」コンテンツをXで発信

製品宣伝ではなく、ターゲットユーザーに価値を提供する。

**Mattia式コンテンツ戦略:**
- 「〇〇してほしい人、コメントして。無料でやります」
- 製品で作った成果物を共有（製品名は出さない）
- 業界のトレンドに関する有益な解説

## まとめ：Sleekの成功方程式

Sleekの成功は、以下の要素の掛け合わせだ。

```
成功 = 過去の蓄積 × 成長市場 × ニッチ問題 × バイラル戦略 × 明確なICP
```

**一見「運が良かった」ように見える成功にも、再現可能な戦略がある。**

重要なのは「最初のプロダクト」で成功しようとしないこと。Mattiaも4つのプロダクトを経てSleekにたどり着いた。

> 「待っているだけでは何も変わらない」

ソロ開発者として、今日から「次に使える」前提でコードを書き、「強い問題」を探し、「価値提供」を発信しよう。
