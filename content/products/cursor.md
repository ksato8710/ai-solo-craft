# Cursor

**Cursor**は、VS Codeをフォークし、**AIをゼロから前提に設計し直したコードエディタ**。見た目はVS Codeだが、中身は別物。2024年に$1B ARRに到達し、2026年2月には評価額$29.3Bに達した。

## 2026年2月24日の大型アップデート

CNBCが報じた最新アップデートの主要ポイント：

### 1. 並列VMエージェント

エージェントが**独自の仮想マシン上で並列実行**可能に。ローカルマシンのリソースを消費せず、10〜20タスクを同時実行できる。

> 「1〜3つのことを同時に実行するのではなく、10〜20のタスクを並列で回せる」
> — Alexi Robbins（Cursor共同エンジニアリングヘッド）

### 2. セルフテスト機能

エージェントが自分の変更を**テストし、ビデオ・ログ・スクリーンショットで記録**できるようになった。

### 3. マルチプラットフォームトリガー

- モバイル
- Slack
- GitHub

### 4. 内部実績

**35%のPull Request**がVM上で動作するエージェントによって生成されている（Cursor社内実績）。

## 特徴

### オートコンプリート（Tab）

業界最高レベルのコード補完。タイプ中に次のコードを予測し、Tabで確定。他社製品と比較して「Cursorなしでは戻れない」という評価が多い。

### Composerモード

自然言語でタスクを指示し、複数ファイルにまたがる変更を自律的に完了。

### @メンション

`@file`、`@folder`、`@web`、`@docs`でコンテキストを明示的に指定。AIが参照すべき情報をコントロールできる。

### マルチモデル対応

Claude、GPT-4、独自モデルを切り替えて使用可能。タスクに応じて最適なモデルを選択できる。

## ユースケース

### 向いている作業

- **日常のコーディング**: オートコンプリートによる高速開発
- **マルチファイル編集**: Composerで複数ファイルを一括変更
- **コードレビュー支援**: AIによる説明・改善提案
- **学習**: コードの意味を質問しながら理解

### 向いていない作業

- **超大規模リファクタリング**: Claude Codeの方が適切
- **JetBrains環境**: プラグイン未提供
