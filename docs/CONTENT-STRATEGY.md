## コンテンツの目的

**「毎日まずこのサイトを見よう」と思わせる情報の網羅性と鮮度を確保する。**

---

## コンテンツ種類別設計（5種類）

### 2026-02-11時点の分類方針（canonical）

- トップ分類は **3種類**:
  - `news`
  - `product`
  - `digest`
- `digest` は `morning` / `evening` を edition として管理
- `dev-knowledge` / `case-study` / `product-update` はニュースのタグとして管理

### 🗞️ 朝夕のまとめ（Digest: morning / evening）

- **配信:** 毎日（朝/夕）
- **内容:** 重要ニュースの **一覧化 + ランク付け（最大Top 10）** を行い、上位 **Top 3** をピックアップして深掘り解説
- **必須要件:**
  - 掲載ニュースをNVA（ニュースバリュー評価）でスコアリングし、順位を明示する
  - Top 3は「背景・数字・なぜ今・ソロビルダー視点」まで踏み込む
  - プロダクトに関するニュースは、必ず該当の **プロダクト記事** にリンクする
    - 存在しない場合はプロダクト記事を新規作成し、以降は更新して使い回す
  - Digest公開ごとに `/news-value` のランキングを更新する（そのウィンドウのTop 10を表示）
- **構成（推奨）:**
  - 重要ニュース一覧（順位 / NVA / 1行要約 / 出典 / 関連プロダクト）
  - Top 3ピックアップ（各ニュースを詳述）
  - 今日のアクション（読者が「今すぐやること」）

### 📰 ニュース（個別）

- **対象:** Digestで扱った重要ニュースのみ（原則Top 3。全ニュースを個別記事化する必要はない）
- **内容:** 1ニュース = 1記事。背景・数字・コミュニティ反応を整理し、ソロビルダー視点で意味付けする
- **必須要件:**
  - NVA評価（スコア表+所見）を掲載
  - プロダクトに関するニュースは、必ず該当の **プロダクト記事** にリンク（なければ新規作成）

### 🏷️ プロダクト（辞書）

- **役割:** 他コンテンツの前提となる「基礎情報」の蓄積。原則、プロダクトごとに **1本の恒久ページ** を持ち、状況変化に応じて更新する
- **運用:**
  - ニュース/事例/ナレッジで登場したら、プロダクト記事の有無を確認
  - 無ければ作成、あれば必要に応じてアップデートしてリンクする

### 📊 ソロビルダー事例紹介（Case Study）

- **内容:** 成功/失敗事例の詳細分析（戦略、収益モデル、意思決定、再現性）
- **必須要件:**
  - 事例に登場するプロダクトは、必ず該当の **プロダクト記事** にリンク（なければ新規作成）

### 🧠 AI開発ナレッジ（Knowledge）

- **内容:** AI開発の実務知見・手順・比較。読者の「明日から使える」再現性を重視
- **必須要件:**
  - 記事内で扱うプロダクトは、必ず該当の **プロダクト記事** にリンク（なければ新規作成）

## カテゴリ統合ルール（現行→方針）

コンテンツの`category`は最終的に以下へ統合する:
`morning-summary` / `evening-summary` / `news` / `dev-knowledge` / `case-study` / `products`

現行で混在していたカテゴリは、次のルールで寄せる:
- `morning-news` → `morning-summary`
- `evening-news` → `evening-summary`
- `product-news` → `news`
- `knowledge` → `dev-knowledge`
- `dev` → `dev-knowledge`
- `deep-dive` → `dev-knowledge`
- `featured-tools` → `dev-knowledge`
- `tools` → `news`（ただし中身が「プロダクト辞書」相当なら `content/products/` に移す）

## 記事品質基準

### 必須要件（全記事共通）

- [ ] タイトルに「何が起きたか」or「何を得るか」が含まれている
- [ ] 定量データ（MAU/調達額/Stars等）が最低1つ含まれている
- [ ] サービス紹介時に開始時期が明記されている
- [ ] 出典リンクが明記されている
- [ ] 「なぜ今盛り上がっているか」の文脈がある
- [ ] NVA評価セクション（DigestのTop 3、個別ニュース）
- [ ] 登場プロダクトの内部リンク（/products/[slug]）がある（無ければ作成する）
- [ ] frontmatter が正しい（title, slug, date, category, description, readTime）

### 海外記事の紹介ルール

- **形式:** 要点紹介 + 日本市場への独自分析
- **禁止:** 全文翻訳の転載
- **必須:** 出典リンク、原文へのリスペクト
- **付加価値:** ソロビルダー視点での実現可能性・注意点
- **表記:** 「本記事は翻訳ではなく、原文の紹介・解説記事です」

## 内部リンク戦略

- Digest → Top 3の個別ニュース、関連プロダクトへリンク
- ニュース（個別）→ 関連プロダクトへリンク
- 事例/ナレッジ → 関連プロダクトへリンク
- プロダクト → 関連ニュース/事例/ナレッジへの逆リンク（可能なら）

## SEOキーワード戦略

### 主要キーワード群

1. **ツール名 + レビュー/使い方:** 「Claude Code 使い方」「Cursor 比較」
2. **カテゴリ + おすすめ:** 「AIコーディングツール おすすめ」「AI画像生成 比較」
3. **課題 + 解決:** 「個人開発 収益化」「AIで稼ぐ方法」
4. **人名 + 戦略:** 「Pieter Levels 戦略」「ソロファウンダー 成功事例」
5. **ニュース + 解説:** 「OpenAI 最新 2026」「AI ニュース 今日」
