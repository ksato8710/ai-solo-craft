# ニュースレター法務・配信ガードレール

最終更新: 2026-02-19

このドキュメントは、AI Solo Craft のニュースレター配信で必ず守るべき法務・配信・編集上の最低基準を定義する。

## 1. 法務ガードレール

### 1-1. 特定電子メール法（日本）
- 事前同意（オプトイン）なしで広告・宣伝メールを送らない。
- 送信者情報（事業者名・連絡先）を明示する。
- 配信停止方法をわかりやすく提供する。
- 配信停止依頼を受けた宛先には再送しない。

### 1-2. 引用・著作権
- 他メディア本文の転載を禁止。
- 引用は必要最小限に限定し、主従関係を守る。
- 「要約・解説」を基本とし、原文の再配布にならない構成にする。
- すべての記事で一次情報リンクを明示する。

### 1-3. 広告・タイアップ表示
- スポンサー起点の話題は `PR` / `Sponsored` を明示する。
- 編集記事と広告枠を視覚的に分離する。

## 2. 配信ガードレール

### 2-1. ドメイン認証
- SPF / DKIM / DMARC を設定し、`Verified` ドメインのみで送信する。
- `From` ドメインと実運用ドメインを一致させる。

### 2-2. 解除導線
- One-Click Unsubscribe を有効化する。
- `List-Unsubscribe` / `List-Unsubscribe-Post` ヘッダーを必ず付与する。
- 配信停止トークンは購読者ごとに一意にする。

### 2-3. 送信品質
- ハードバウンス/苦情アドレスは再送禁止にする。
- 送信ログ（成功/失敗/エラー詳細）を保持する。
- 同日二重配信を禁止する。
- 件名と本文の主旨不一致を禁止する（クリック誘導型の誇張件名禁止）。

## 3. 編集ガードレール（検知レイヤー運用）

### 3-1. ソースの役割分離
- ニュースレター: 検知専用（detector）
- 一次情報（公式発表）: 事実確定（verifier）
- 日本メディア: ローカライズと国内文脈補足（localizer）

### 3-2. 記事化条件
- 1トピックにつき最低2リンクを必須化する。
  - EN一次情報リンク（必須）
  - JPリンク（一次情報がなければ信頼できる国内メディア）
- 一次情報がない話題は「未確定」と表示し、断定しない。

### 3-3. 言い回し
- 推測は「推測」「未確認」と明示する。
- 数字（コスト、性能、日付）は出典リンク付きで記述する。

## 4. 競合ニュースレター収集ガードレール

- 初期受信口は `ktlabworks@gmail.com` の単一構成とする。
- 受信実績は `source_delivery_observations` に記録し、未受信監視を行う。
- 競合ニュースレター本文は「検知用途」に限定し、転載・再配布しない。
- Gmailの受信箱から削除せず、必要時にトレース可能な状態を維持する。

## 5. 配信前チェック（必須）

- [ ] `From` ドメインが Verified である。
- [ ] One-Click Unsubscribe が有効。
- [ ] 宛先は active subscriber のみ。
- [ ] 同日配信済みチェックを通過。
- [ ] 各トピックに EN一次 + JPリンクの2本以上を付与。
- [ ] PR枠の明示（運用開始後）。
- [ ] 送信ログ保存（成功/失敗/エラー）。
