# EDITORIAL.md — AI Solo Builder 編集ガイドライン

## ニュースタイトルの原則（最重要）

**すべてのニュース記事タイトルには「何が起きたか」を必ず含めること。**

読者が一目で「このニュースは何か」を理解できないタイトルは不合格。

### 構造
```
[主語] + [何が起きたか] — [一言の文脈・意味]
```

### ❌ 悪い例
- `Perplexity — AI検索エンジン`（何のニュース？）
- `ElevenLabs — AI音声生成プラットフォーム`（何が起きた？）

### ✅ 良い例
- `Perplexity、評価額$20Bに到達 — AI検索でGoogle対抗の本命`
- `ElevenLabs、Series C $180Mで評価額$3.3B — AI音声生成の覇者`

### パターン別
- **資金調達:** `[社名]、[金額]調達（評価額$XX） — [意味]`
- **新機能:** `[プロダクト] [バージョン]リリース — [何が変わった]`
- **買収:** `[買収企業]、[被買収企業]を買収 — [業界への意味]`
- **急成長:** `[社名]、ARR $XX達成 — [なぜ伸びている]`
- **論争:** `[社名]、[何が問題] — [影響]`

### オリジナルコンテンツ（ニュースではない記事）
- 「何が起きたか」ではなく「読者が何を得るか」をタイトルに
- 例: `Claude Codeで稼ぐ15のビジネスアイデア｜月3〜50万円の収益モデルを日本向けに解説`

## 点検時チェックリスト

記事の品質チェック時に、以下を確認:

- [ ] タイトルに「何が起きたか」が含まれているか
- [ ] 読者がタイトルだけで記事の価値を判断できるか
- [ ] プロダクト名だけのタイトルになっていないか
