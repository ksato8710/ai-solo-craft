# 競合ニュースレター収集運用（最小構成）

最終更新: 2026-02-21

## 0. 目的

競合ニュースレターの「実際の配信レター」を確実に収集し、
`AI Solo Craft` 向け朝刊統合の検知レイヤーとして使う。

初期フェーズは **単一受信口** で開始する。

- 受信口: `ktlabworks@gmail.com`
- 収集単位: 実際に受信した1通ごとの観測ログ

## 1. 最小インフラ構成

1. 受信
- すべての競合ニュースレターを `ktlabworks@gmail.com` で購読する。

2. Gmail側整理
- ラベルを作成する。
  - `nl/competitor`
  - `nl/competitor/tier-a`
  - `nl/competitor/tier-b`
- フィルタを作成する。
  - 条件: `from:(news@daily.therundown.ai OR ... )`
  - 操作: `Skip Inbox` は使わず、必ず受信箱に残す。
  - 操作: 対応ラベルを付与。

3. 観測ログ化
- 管理画面 `/admin/collection` で受信ログを記録する。
- 必須入力:
  - ソース
  - 受信日時
  - 件名
  - Fromアドレス
- 推奨入力:
  - Read Online URL
  - Message-ID
  - Gmail Labels

4. スケジュール突合
- `/admin/schedules` でニュースレター到着想定時刻を管理。
- `/admin/collection` で過去24時間未受信を監視する。

## 2. 収集ルール

- 競合ニュースレター本文の転載はしない（要約のみ）。
- 収集したレターは detector として扱う。
- 記事化時は必ず verifier（一次情報）を付与する。
- Top3に採用する話題は localizer（日本語補足）も必須。

## 3. 日次運用フロー

1. 朝の収集
- `ktlabworks@gmail.com` の `nl/competitor` を確認。
- 新着を `/admin/collection` に記録。

2. 欠落チェック
- `/admin/collection` の未受信アラートを確認。
- 欠落があれば配信遅延か配信停止かを確認。

3. 統合準備
- 検知トピックを抽出。
- 一次情報リンクと日本語補足リンクをセットで付与。

## 4. 受信漏れ時の対応

- 24時間未受信が続く場合:
  - Gmailフィルタ誤設定を確認。
  - 送信元メールアドレス変更の有無を確認。
  - 競合ニュースレターの購読状態を確認。

## 5. 次フェーズ（必要になったら）

- Gmail API連携による自動取り込み
- 複数受信口（Gmail + 独自ドメイン）で冗長化
- HTML/プレーンテキスト本文の自動解析

現時点では追加しない。まずは単一受信口の安定運用を優先する。
